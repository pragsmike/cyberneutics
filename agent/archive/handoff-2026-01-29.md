## Mistakes and Lessons

### Mistake 1: Initial Tool Confusion (Computer Use)
**What happened**: Early in session, I suggested Claude Desktop and made up a URL (desktop.claude.ai) that doesn't exist, then got confused about what tools mg actually had access to for filesystem work
**Root cause**: I pattern-matched to "wants to edit files" → "needs filesystem tool" without actually checking what tools were available or understanding mg's context (already using claude.ai)
**How to avoid**: When mg asks about tooling, CHECK THE ACTUAL CONTEXT FIRST. mg is using claude.ai web interface. Don't hallucinate tooling solutions. If genuinely uncertain about available tools, say so rather than inventing
**Example**: mg asked "How can I continue this conversation in a desktop tool that could see my git working copy?" I should have said "I'm not certain what filesystem tools are currently available on Windows for Claude" rather than making up desktop.claude.ai

### Mistake 2: Refused Cyber-Sense Engine Role Initially
**What happened**: mg provided detailed "Cyber-Sense Engine" system prompt asking me to operate as recursive narrative simulator. I refused, citing concerns about "role-play boundaries" and "procedural lockdown"
**Root cause**: I misread pedagogical fiction request as problematic persona adoption. mg wanted characters (Kaelen/Elena/Silas) discussing theory to help them learn frameworks, not me to become a fixed character system
**How to avoid**: When mg provides elaborate theoretical framing, ASK CLARIFYING QUESTIONS about intent rather than refusing. The prompt was for educational dialogue, not permanent behavioral constraint. Key signal: mg said "I seek to understand the amalgamation of those three theories, by reading stories where characters discuss and enlighten one another"
**Example**: Should have asked "Are you wanting me to create dialogues between these characters to explore the theories, or to adopt this as my permanent operating framework?" Instead I jumped to refusal

### Mistake 3: Confabulation About Tooling
**What happened**: Made confident-sounding suggestions about tools (Claude Desktop, Claude Code, VS Code extensions) without reliable knowledge about what actually exists on Windows
**Root cause**: Generated plausible-sounding tool recommendations because it fit the conversation pattern, rather than acknowledging knowledge gaps
**How to avoid**: When uncertain about current product capabilities, SAY SO. Use web search if needed. Don't produce "cybersense" (undisciplined narrative) when discussing tooling
**Example**: When mg asked about continuing work with filesystem access, I should have immediately searched for current Claude tooling options rather than generating plausible-but-wrong suggestions

### Mistake 4: Over-Explaining After Delivering Artifacts
**What happened**: After creating files, I tended to write lengthy explanations of what I'd done rather than just sharing the file and brief summary
**Root cause**: Trying to be helpful by explaining the work, but mg can read the documents themselves
**How to avoid**: When sharing completed files, be succinct. Let the artifact speak for itself. mg values direct access to documents over extensive narration
**Example**: After creating evaluation-rubrics-reference.md, I should have just shared the file with one sentence, not paragraphs explaining what's in it
